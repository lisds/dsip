Time and frequency#

The Fourier transform can take the values of a 1D array, and recast these values as weights for sinusoids.

The key thing to understand in the Fourier transform is the idea of rewriting the input 1D signal as a series of weights for basis functions. We will start with some very trivial basis functions, and then move on to the much less trivial Fourier basis functions, which are sinusoids.

import numpy as np
np.set_printoptions(precision=2)
import pandas as pd
pd.set_option('mode.copy_on_write', True)
import matplotlib.pyplot as plt
# Set default colormap.
plt.rc('image', cmap='grey')

The data are from the first training file in a single-lead ECG dataset. We select the first 512 samples. The sampling rate was 300 Hz, so this is just a bit less than 2 seconds of data.

We first illustrate the idea of recasting the data as basis functions and weights, just using the first 10 values in the array.

ecg_arr = np.array(ecg['Amplitude'])

The key idea is that we can think of these 10 observations as being the weighted sum of some basis functions.

First we will make a very dumb set of basis functions. These basis functions are called delta functions because they have one value that is 1 (the delta) and the rest of the values are 0.

Finally, we reconstruct the original signal by summing our multiplied basis functions along the columns (across rows):

reconstructed = multiplied.sum(axis=0)
reconstructed

You can do the multiplication and sum procedure in one stop with the dot-product procedure. See vectors and dot products for a description of the dot product idea, and, and see the links at the top of this PCA tutorial for helpful resources to explain how the dot product works on 2D arrays - to become matrix multiplication.

# Reconstructing from a basis with the dot product:
basis_functions.dot(first_10)

The Fourier basis#

So far our basis functions have been silly - but the Fourier transform uses more interesting basis functions.

To summarize, the Fourier transform finds sinusoids as basis functions, and weights for each sinusoid. It can then express the data as a weighted sum of the sinusoids. The weights express the amount of that sinusoid in the original signal.

Let’s start by running a Fourier transform on the data.

ft_ecg = np.fft.fft(ecg_arr)
# Show the first 10 coefficients (weights)
ft_ecg[:10]

array([-35205.      +0.j  , -66237.69+21181.04j, -26006.78-31944.24j,
         5422.84-17439.78j,  10207.57 -5222.19j,  14682.31-16536.32j,
         3039.51 -5803.01j,  -5717.88+14246.36j,  11728.84 -3619.9j ,
        10580.39 -5078.49j])

Notice that the resulting weights are complex numbers. This is one way of expressing both the phase of the sinusoid (the left-right shift of the cosine wave), and the amplitude of the sinusoid. The first weight corresponds to a sinusoid of frequency 0 (no cycles), the next to a sinusoid of frequency 1 (one cycle over the whole 1D array), and so on.

# The number of basis functions the Fourier transform has used.
# This is also the number of sinusoids needed to reconstruct the original data.
n_ft_bases = len(ft_ecg)
n_ft_bases

512

# The number of samples in the original data.
n

512

Rather than go into the complex numbers, we get the amplitude and phase information from the complex number weights:

# One amplitude for each sinusoid.  `np.abs` does this calculation on complex numbers.
amplitudes = np.abs(ft_ecg)
amplitudes[:10]

array([35205.  , 69541.84, 41192.07, 18263.44, 11465.85, 22113.8 ,
        6550.84, 15350.99, 12274.74, 11736.09])

# One phase (angle) for each sinusoid.
angles = np.angle(ft_ecg)
angles[:10]

array([ 3.14,  2.83, -2.25, -1.27, -0.47, -0.84, -1.09,  1.95, -0.3 ,
       -0.45])

See https://matthew-brett.github.io/teaching/fourier_basis.html

The Fourier transform is therefore using all these basis functions - each basis function is a sinusoid. The sinusoids become faster and faster in frequency as we go from first to last:

ft_basis = np.zeros((n_ft_bases, n))
ns = np.arange(n)
one_cycle = 2 * np.pi * ns / n
for k in range(n_ft_bases):
    # Make an input to cos for this frequency.
    t_k = k * one_cycle
    # Use the angle to shift the phase of the sinusoid.
    ft_basis[k, :] = np.cos(t_k + angles[k])

Show the first 10 sinusoids:

fig, axes = plt.subplots(10, 1)
for i in range(10):
    axes[i].plot(ft_basis[i])
axes[-1].set_xlabel('Samples')
fig.suptitle('First 10 Fourier transform sinusoid bases');

Remember, the Fourier transform has re-expressed the original data as the weighted sum of this collection of sinusoids (basis functions). To reconstruct the original data, we multiply the sinusoid (basis function) by it’s corresponding weight, and sum the result over the columns (over weighted sinusoids).

ft_multiplied = ft_basis * amplitudes[:, None]
